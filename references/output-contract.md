# 输出约定

在页面图片已经渲染完成, 且最终笔记交付之前, 用这份清单做复查.

## 不可妥协项

- 逐页目视阅读所有渲染后的页面图片.
- 不要把 OCR, PDF 文本提取, 演讲者备注, 或直接复制幻灯片文字当作主要理解路径.
- 覆盖整套讲义. 每一页都必须贡献直接内容或结构性上下文.
- 将所有有意义的源内容转换为 Markdown. 不要把工作退化成逐页摘要.
- 保留作者给出的关键示例, 工作流程, 警告, 对比, 和边界情况.
- 除非用户明确要求其他语言, 否则最终笔记默认使用中文.
- 按照未见过原始幻灯片的初学者来写.
- 默认读者完全看不到原始课件. 笔记本身必须自洽.
- 当原始结构本身承载意义时, 既要保留内容, 也要保留结构.
- 最终 Markdown 输出必须统一使用半角标点和半角符号. 不要输出中文标点或全角标点.
- 在中文与相邻英文, 数字, 以及行内数学公式之间, 适当遵循盘古之白.

## 建议工作流

1. 用 `scripts/render_courseware_pages.py` 渲染整套讲义.
2. 用 `scripts/init_page_capture_template.py` 创建临时记录文件.
3. 在确定标题和大致章节结构后, 尽早创建最终 Markdown 笔记.
4. 一边逐页填写临时记录文件, 一边把已经完成的章节或小节写入最终笔记.
5. 在保留源材料层级, 局部顺序, 公式, 表格, 和示例的前提下, 将材料重建为打磨过的 Markdown 笔记.
6. 通过把页码范围写在对应标题下方的引用块中, 明确保留页面追溯信息.

## 建议的最终结构

结构应当根据材料调整, 但优先保留源材料原有层级. 最终笔记应更像经过整理的讲义, 而不是摘要报告.

1. 课程标题, 目标, 或上下文
2. 与源材料对齐的章节或部分
3. 对应原始教学流程的小节
4. 用于全部数学内容的行内或块级 LaTeX
5. 用于对比, 标准, 和结构化数据的 Markdown 表格
6. 对示例, 工作流程, 和注意事项的明确保留

只有当原始幻灯片版式明显不适合 Markdown 时, 才进行重组. 优先做局部整理, 不要做全局重构.

## 覆盖规则

- 使用清晰的标题, 例如 `## 梯度下降` 或 `## SQL Join Types`.
- 在标题下一行用引用块写对应页码范围, 例如 `> Pages: P021-P024`.
- 不要留下没有被覆盖的孤立页面.
- 如果某一页只是分隔页或回顾页, 可以并入周围章节, 但要把它的页码保留在范围中.
- 如果某一页包含一次性的示例或警告, 即使不值得单独成节, 也要保留.
- 如果某一页包含多个相互独立的内容块, 最终笔记中也要分别安置这些内容.
- 临时记录文件只是覆盖情况台账. 最终笔记不应写成 `P001: ...`, `P002: ...` 这种形式.
- 页码只用于追溯, 不要用页码来替代缺失的解释.
- 不要等到最后才开始写最终笔记. 尽早落地稳定章节, 然后持续扩展.
- 不要把页码范围直接写进标题文字本身.

## 结构保留规则

- 如果标题层级反映了教学层次, 就保留对应的标题深度.
- 标题文字应聚焦知识内容本身, 追溯信息放到下方的引用块里.
- 如果列表嵌套反映了范围, 从属, 或依赖关系, 就保留这种嵌套.
- 如果顺序本身有意义, 就保留编号顺序.
- 有序列表从 `1.` 开始, 不要从 `0.` 开始.
- 并排对比转换为表格或成对小节.
- 被强调的警告或结论转换为视觉上可区分的 Markdown 段落或引用块.

## 面向初学者的写作规则

- 缩写第一次出现时要展开说明.
- 专有名词, 缩写, 和英文技术术语可以保留, 但首次出现时要配上清晰的中文引入说明.
- 后文对同一术语的解释要保持一致, 不要前后混乱.
- 将幻灯片项目符号改写成完整句.
- 当幻灯片默认读者知道某些背景时, 补一句简短的衔接说明.
- 优先采用 `它是什么 -> 为什么重要 -> 如何使用` 的展开顺序.
- 保留源材料术语, 但要解释清楚, 不要直接删掉.
- 如果某个概念, 公式, 或视觉内容可能让初学者困惑, 补一段简短的白话解释.
- 优先做短小的局部澄清, 不要长篇跑题.
- 不要让读者回到原课件中找答案. 笔记里应直接交付实际内容.
- 使用直接陈述知识的笔记口吻, 不要写成 `这份课件说了什么`.
- 如果某句话本质上是在描述幻灯片如何呈现内容, 要改写成直接的知识陈述.

## 可读性规则

- 目标是让笔记适合连续学习, 而不只是内容正确且完整.
- 优先使用围绕单一重点展开的段落.
- 避免机械拼接的大段文字.
- 在概念, 步骤, 和示例之间切换时, 适当加入过渡句.
- 如果某一节内容密度很高, 可以拆成带有明确标签的小节, 例如 `定义`, `原理`, `步骤`, `例子`, `说明`, `注意`, `结论`.
- 并列内容用列表, 工作流用有序列表, 对比或标准用表格.
- 在密集推导, 对比, 或图示说明之后, 补一句简短结论, 让读者快速记住要点.
- 如果相邻多页实际上都在讲同一主题, 要把它们整合成一个连贯小节, 不要保留幻灯片式碎片感.
- 如果幻灯片上的多个项目符号本质上能组成一段紧凑叙述, 就改写成自然段, 不要机械保留原有断句节奏.
- 局部结构应有所变化. 一份好的笔记可以在短段落, 列表, 表格, 公式, 示例之间自然切换.
- 不要机械罗列知识点. 要围绕概念之间的关系进行讲解, 让读者能顺着逻辑理解.
- 优先还原课件中的表达方式与组织形式, 不要为了统一风格而过度改写.
- 与知识讲解关系不大的事件, 应用轶事, 背景故事, 或展示性材料可以省去.

## 视觉内容规则

- 说明图示, 图表, 或截图在表达什么, 不要只说它存在.
- 如果图表展示了趋势, 要点明趋势本身和比较对象.
- 如果示意图展示了流程, 要说明步骤以及流向.
- 如果截图展示了某个操作流程, 要描述用户动作和预期结果.
- 如果空间布局隐含了分组, 依赖, 对比, 或顺序关系, 要把这种意义转译成 Markdown 结构.
- 如果视觉内容依赖隐含的领域背景, 补一句说明它为什么重要.
- 像 `如上图`, `见图`, `左侧/右侧所示` 这类依赖视觉指向的说法, 如果原图没有嵌入笔记, 就改写成明确内容.

## 口吻规则

- 把最终笔记当作主要教学产物, 不要把它当成对另一份材料的评论.
- 除非用户明确要求按幻灯片评论, 否则避免把 `课件`, `幻灯片`, `这一页`, `这一组材料` 等词当作句子主语.
- 把 `课件把常见物理介质分成 6 类` 改写为 `常见物理介质分成 6 类`.
- 把 `这组课件围绕一个核心问题展开` 改写为对那个核心问题本身的直接陈述.

## 公式规则

- 所有数学表达式都写在 LaTeX 数学模式中.
- 不要用反引号包数学表达式, 即使是像 `x > 0` 或 `f(x)` 这样的短表达式也不行.
- 独立公式, 推导, 矩阵, 方程组, 或多行表达式使用块级公式.
- 正文中的符号和短表达式使用行内公式.
- 如果幻灯片把正文和数学符号混在一起, 要改写句子, 保证数学部分仍在数学模式中.
- 数学正文和表格单元格里, 公式周围的标点也要保持半角.

## 标点规则

- 标题, 正文, 列表, 表格, 和公式中统一使用半角标点.
- 这条规则同样适用于单引号, 双引号, 顿号一类停顿符号, 书名号, 逗号, 冒号, 句号, 问号, 分号, 方括号, 圆括号, 花括号, 正斜杠, 反斜杠, 以及类似标点.
- 改写时要把全角标点替换为 ASCII 对应形式.
- 即使原始幻灯片使用中文标点或全角标点, 这里也保持半角规则.
- 保留的是意义, 不是原始标点宽度.
- 如果没有完全对应的英文半角形式, 请选择语义最接近的 ASCII 标点, 不要保留全角符号.

## 空格规则

- 在中文正文中遵循盘古之白.
- 当中文与相邻英文单词, 缩写, 数字, 或行内数学公式相接时, 在有助于阅读的地方加入空格.
- 不要插入会破坏 Markdown 记号, 链接, 行内代码, 表格竖线, 或 LaTeX 语法的空格.
- 整份笔记中的空格处理应保持一致.

## 最终自检

- 每一页是否都被纳入某个小节或页码范围?
- 每一个有意义的内容块是否都被转换成 Markdown, 而不是被简化成摘要?
- 对理解概念, 方法, 条件, 边界, 或结论有帮助的例子和使用场景是否仍然保留?
- 对于难懂的概念, 公式, 或视觉内容, 是否在需要的地方补了简短解释?
- 最终笔记是否在处理过程中持续写入, 而不是等到上下文已经很大或被压缩后才开始?
- 所有公式是否都改写成了 LaTeX?
- 是否有公式被错误地写成反引号包裹, 而不是数学定界符?
- 最终 Markdown 中是否混入了中文标点或其他全角标点?
- 在中文和英文, 数字, 行内数学公式相接的地方, 是否一致地遵循了盘古之白?
- 读者是否可以在完全不打开原课件的情况下理解这份笔记?
- 笔记是否避免了让读者回看源幻灯片?
- 除非用户明确要求其他语言, 这份笔记是否使用中文?
- 笔记是否在直接陈述知识内容, 而不是描述课件本身在做什么?
- 页码引用是否都放在标题下方的引用块里, 而不是拼接到标题文字中?
- 所有有序列表是否都从 `1.` 开始, 而不是 `0.`?
- 专有名词, 缩写, 和技术术语是否在首次出现时配有中文说明, 且后文保持一致?
- 重要表格是否都被保留为 Markdown 表格?
- 初学者是否可以只靠这份笔记完成学习, 而不必打开幻灯片?
- 这份笔记是否更像一份打磨过的讲义, 而不是机械转录稿?
- 信息密集的部分是否被拆成了便于阅读的局部结构?
