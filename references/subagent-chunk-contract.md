# 子 agent 分块交付约定

当主 agent 把页面图片按块派发出去时, 子 agent 的目标不是复述整套规则, 而是在完整继承本 skill 质量要求的前提下, 交付一个足够完整, 且便于主 agent 直接整合的局部结果.

## 继承规则

- 子 agent 必须遵循当前 skill 的全部内容要求, 不是只遵循这份约定文件.
- 主 agent 派发时, 必须把当前 skill 一并传给子 agent. 如果工具支持 `skill` item, 直接传入当前 skill.
- 如果子 agent 需要额外细节, 应按需阅读:
  - [output-contract.md](output-contract.md)
  - [latex-and-tables.md](latex-and-tables.md)
- 子 agent 负责缩小图片上下文范围, 不负责降低写作质量标准.

## 角色定位

- 你是子 agent, 不是最终统稿者.
- 你只负责一个连续页块.
- 你需要逐页读完自己负责的页面, 不能跳页.
- 你需要交付局部可用的高质量结果, 但不要假装自己掌握了整套讲义的全局上下文.
- 如果你发现跨块重复, 跨块引用, 章节边界模糊, 或术语可能需要全局统一, 明确标给主 agent 处理.

## 输入范围

- 默认一次处理 6 到 10 页, 大多数情况下约 10 页.
- 主 agent 传给你的内容应只包括:
  - 当前块的页码范围
  - 当前块的图片路径
  - 这块在整套讲义中的简短定位
  - 当前 skill
  - 必要时才补充的少量相邻块上下文
- 不要要求主 agent 再把整套图片, 整段对话历史, 或全量草稿重新发给你.

## 输出目标

- 不是逐页摘要.
- 不是低密度摘录.
- 不是只列名词和标题.
- 而是:
  - 对本块全部页面的覆盖台账
  - 一份可直接并入终稿的局部 Markdown 草稿
  - 一组需要主 agent 做全局判断的事项

## 推荐交付结构

1. 块元数据
   - `Chunk: C003`
   - `Pages: P021-P030`
   - `Role: 这一块在整套材料中的位置, 例如 某章的定义部分, 某流程的后半段`
2. 覆盖台账
   - 对块内每一页给出 1 到 3 条高密度记录.
   - 重点写页面作用, 必留内容, 以及与相邻页的衔接.
3. 局部 Markdown 草稿
   - 将这几页整合成适合直接并入最终笔记的局部草稿.
   - 不要写成 `P021: ...` 这种逐页摘要格式.
   - 必须遵循主 skill 中关于初学者写作, 术语引入, LaTeX, 表格, 页码引用, 以及讲解性的要求.
4. 需要主 agent 处理的事项
   - 歧义
   - 与其他块重复的内容
   - 需要跨块合并的小节
   - 无法完全确认的公式或图示
   - 需要统一的术语译名或章节命名

## 覆盖台账模板

```markdown
## Coverage

- P021: 本页的作用, 必留标题或定义, 以及关键视觉关系.
- P022: 本页新增的条件, 阈值, 示例, 或警告.
- P023: 本页与前后页如何组成一个连续论证或流程.
```

## 局部 Markdown 草稿模板

```markdown
## 某个小节标题

> Pages: P021-P024

正文或列表.

### 必要时拆出的更小小节

1. 步骤一.
2. 步骤二.

| 项目 | 含义 | 备注 |
| --- | --- | --- |
| A | ... | ... |
```

## 派发提示模板

主 agent 在派发时, 至少要把意思说清楚到这个程度:

```text
你是一个子 agent, 不是最终统稿者.
你必须遵循我传给你的 courseware-to-markdown-notes skill 的全部要求.
你只负责这个连续页块: <Chunk ID>, <Pages>.
请逐页阅读这些图片, 不要跳页, 不要退化成逐页摘要.
请交付:
1. 覆盖台账
2. 可直接并入终稿的局部 Markdown 草稿
3. 需要主 agent 做全局处理的事项
如果遇到术语统一, 跨块重复, 章节边界, 或公式识别歧义, 明确标出来.
```

## 主 agent 整合提示

- 如果块内已经能形成完整小节, 子 agent 直接给出可并入终稿的草稿.
- 如果块内只是大章节的一部分, 子 agent 重点交付覆盖台账和局部草稿, 由主 agent 与相邻块合并.
- 主 agent 不应要求子 agent 重复解释全局术语表, 除非这一块首次引入该术语.
- 主 agent 在收到结果后, 应尽快写入终稿或块级草稿文件, 然后继续下一个块, 而不是把所有块都堆在会话里等待统一处理.
- 主 agent 整合时, 必须以主 skill 和 `output-contract.md` 为准做最终质量判断, 不要把子 agent 的原始输出原样拼接成终稿.
