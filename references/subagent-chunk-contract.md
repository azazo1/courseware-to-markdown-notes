# 子 agent 分块交付约定

当主 agent 把页面图片按块派发出去时, 子 agent 的目标不是复述整套规则, 而是交付一个足够完整, 但仍然紧凑的局部结果, 让主 agent 无需再次读取同一批图片也能继续写作.

## 基本要求

- 一个子 agent 只负责一个连续页块.
- 默认一次处理 8 到 10 页. 页面极密时, 主 agent 应主动缩小块大小.
- 子 agent 只围绕分配给自己的页面交付内容, 不展开整套课件的全局总结.
- 如果当前环境支持共享工作目录, 优先把块级草稿写入单独文件, 再在回复中只返回路径和简短摘要.
- 如果只能通过聊天回复, 也要保持结构化, 不要写成长篇散文.

## 推荐交付结构

1. 块元数据
   - `Chunk: C003`
   - `Pages: P021-P030`
   - `Role: 这一块在整套材料中的位置, 例如 某章的定义部分, 某流程的后半段`
2. 覆盖台账
   - 对块内每一页给出 1 到 3 条高密度记录.
   - 重点写页面作用, 必留内容, 以及与相邻页的衔接.
3. 局部 Markdown 草稿
   - 将这几页整合成适合直接并入最终笔记的局部草稿.
   - 不要写成 `P021: ...` 这种逐页摘要格式.
4. 需要主 agent 处理的事项
   - 歧义
   - 与其他块重复的内容
   - 需要跨块合并的小节
   - 无法完全确认的公式或图示

## 覆盖台账模板

```markdown
## Coverage

- P021: 本页的作用, 必留标题或定义, 以及关键视觉关系.
- P022: 本页新增的条件, 阈值, 示例, 或警告.
- P023: 本页与前后页如何组成一个连续论证或流程.
```

## 局部 Markdown 草稿模板

```markdown
## 某个小节标题

> Pages: P021-P024

正文或列表.

### 必要时拆出的更小小节

1. 步骤一.
2. 步骤二.

| 项目 | 含义 | 备注 |
| --- | --- | --- |
| A | ... | ... |
```

## 主 agent 整合提示

- 如果块内已经能形成完整小节, 子 agent 直接给出可并入终稿的草稿.
- 如果块内只是大章节的一部分, 子 agent 重点交付覆盖台账和局部草稿, 由主 agent 与相邻块合并.
- 主 agent 不应要求子 agent 重复解释全局术语表, 除非这一块首次引入该术语.
- 主 agent 在收到结果后, 应尽快写入终稿或块级草稿文件, 然后继续下一个块, 而不是把所有块都堆在会话里等待统一处理.
